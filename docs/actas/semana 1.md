ACTA DE REUNIÓN SEMANAL
Proyecto: Control de cobros a clientes – @Tequecoso
Modalidad: Asincrónica / Presencial / Virtual
Participantes: Mauricio Núñez
1. Planificación

Se realizó una reunión de planificación para definir el avance correspondiente a la primera entrega del proyecto. Se acordó trabajar el problema de control de cobros a clientes, debido a que actualmente la microempresa registra la información de forma manual en un cuaderno, lo que provoca desorden, atrasos y falta de visibilidad sobre los pagos pendientes.
2. Qué se planificó

Se planificó desarrollar una solución mínima viable que incluya:

    Modelo BPMN AS-IS del proceso actual.

    Modelo BPMN TO-BE con mejoras propuestas.

    Implementación básica del flujo en Flowable.

    Formularios para las tareas principales.

    Avance inicial de Web Services en Java.

    Organización de la evidencia en GitHub.

3. Qué se hizo

Durante la semana se avanzó en la definición del problema y en la construcción de la primera versión de la solución. Se modeló el proceso de cobro a clientes, considerando tareas como registrar deuda, revisar estado de pago, enviar recordatorio y registrar pago. También se comenzó la implementación básica del backend en Java con Spring Boot, creando la estructura inicial del proyecto, un modelo de datos simple y un controlador REST básico.
4. Problemas encontrados

Se presentaron dificultades iniciales en el uso de Flowable y en la organización de la estructura del backend en Java. También fue necesario comprender mejor cómo crear archivos directamente en GitHub y cómo mantener coherencia entre el modelo BPMN, los formularios y el servicio web.
5. Próximos pasos

Los siguientes pasos del equipo serán:

    Completar y revisar el BPMN AS-IS y TO-BE.

    Verificar el funcionamiento del proceso en Flowable.

    Ajustar los formularios del proceso.

    Terminar el avance mínimo del Web Service.

    Subir y ordenar toda la evidencia en GitHub.

    Preparar los videos de avance para la emprendedora y para el profesor.

6. Acuerdos

Se acordó continuar trabajando sobre el proceso de cobros a clientes, priorizando una solución funcional mínima y no una solución completa. Además, se acordó que cada integrante mantenga evidencia de su avance en el repositorio GitHub para demostrar el trabajo realizado.
