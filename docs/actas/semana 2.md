# Acta semanal

## ACTA DE REUNIÓN SEMANAL

**Proyecto:** Tequecoso Cobros  
**Semana:** 2  
**Modalidad:** Asincrónica / Presencial / Virtual  
**Participantes:** Mauricio Núñez  

## 1. Planificación

Durante esta semana se planificó continuar el avance del MVP del proyecto Tequecoso Cobros, ampliando el alcance inicial para no trabajar solo cobros a clientes, sino también clientes, proveedores y pagos a proveedores. El objetivo fue mantener coherencia entre BPMN, Flowable, formularios, backend y documentación del repositorio.

## 2. Qué se planificó

Se planificó realizar las siguientes actividades:

- Ajustar y simplificar los procesos BPMN para que fueran ejecutables en Flowable.
- Crear o revisar los formularios asociados a cada proceso.
- Implementar y probar endpoints REST adicionales en Spring Boot.
- Incorporar manejo básico de clientes, cobros, proveedores y pagos a proveedores.
- Ordenar la documentación del repositorio y actualizar los README.
- Organizar capturas como evidencia del funcionamiento del sistema.

## 3. Qué se hizo

Durante la semana se logró avanzar en varios componentes del proyecto:

- Se dejaron listos procesos funcionales en Flowable para registro de clientes, registro de proveedores, cobro a clientes y pago a proveedores.
- Se revisaron y asociaron formularios a los procesos definidos.
- Se probó la ejecución de los procesos en Flowable Work.
- En el backend con Spring Boot se implementaron endpoints básicos tipo GET y POST para clientes, cobros, proveedores y pagos a proveedores.
- Se realizaron pruebas locales del backend usando terminal con comandos curl.
- Se actualizaron los archivos README del proyecto principal, del backend y de la carpeta Flowable.
- Se ordenó mejor la evidencia dentro del repositorio para facilitar la revisión de la entrega.

## 4. Problemas encontrados

Durante el avance se identificaron algunos problemas:

- Dificultades iniciales para probar endpoints sin herramientas como Thunder Client.
- Necesidad de adaptar la forma de prueba usando curl desde terminal.
- Diferencias entre lo documentado inicialmente y el alcance real que fue creciendo durante el desarrollo.
- Necesidad de limpiar documentación generada con texto no apropiado para la entrega final.

## 5. Próximos pasos

Los próximos pasos definidos fueron:

- Verificar que todas las carpetas del repositorio tengan contenido coherente con su README.
- Mantener evidencia visual de los procesos y formularios en Flowable.
- Preparar la explicación técnica para el video del profesor.
- Preparar una explicación simple y orientada al valor del proyecto para el video de la emprendedora.
- Revisar qué componentes faltan fortalecer para la Entrega 2.
- Evaluar mejoras futuras como persistencia de datos o integración más directa entre Flowable y backend.

## 6. Acuerdos

Se acordó mantener el enfoque MVP, priorizando una solución funcional y demostrable. También se acordó que toda mejora realizada debe reflejarse en el repositorio con evidencia clara, documentación ordenada y coherencia entre BPMN, formularios, backend y objetivos del proyecto.
